# UnoEngine - Task Completion Checklist

タスクが完了したら、以下の項目を確認してください。

## 1. Code Quality

### Build Verification
- [ ] プロジェクトがビルドエラーなしでコンパイルできる
- [ ] 警告（Warnings）がないか確認（可能な限り警告も解決する）
- [ ] Debug/Release両方のビルド設定で確認

### Code Style
- [ ] コードスタイルガイドラインに従っている（`code_style_conventions.md`参照）
- [ ] 命名規則が一貫している
- [ ] 適切な日本語コメントが付いている（複雑なロジックのみ）
- [ ] 不要なコメントアウトされたコードを削除

### Code Structure
- [ ] 適切なヘッダーガードまたは`#pragma once`を使用
- [ ] includeの順序が正しい（対応ヘッダ→エンジンヘッダ→システムヘッダ→標準ライブラリ）
- [ ] 重複コードがない
- [ ] クラス/関数が適切なファイルに配置されている

## 2. Functionality

### Runtime Testing
- [ ] アプリケーションが正常に起動する
- [ ] 実装した機能が期待通りに動作する
- [ ] クラッシュやフリーズが発生しない
- [ ] メモリリークがない（可能な範囲で確認）

### DirectX12 Specific
- [ ] DirectX12のデバッグレイヤーでエラーが出ていない
- [ ] GPUリソースが適切に解放されている
- [ ] コマンドリストの同期が正しく行われている
- [ ] リソースバリアが適切に配置されている

## 3. Documentation

### Code Documentation
- [ ] 新しいクラスや複雑な関数に適切なコメントがある
- [ ] publicインターフェースが明確に理解できる

### Memory Updates
- [ ] 新しい機能や重要な変更がメモリファイルに記録されている
- [ ] プロジェクト構造に変更があれば`project_overview.md`を更新
- [ ] 新しいコマンドやワークフローがあれば`suggested_commands.md`を更新

## 4. Project Files

### Visual Studio Project
- [ ] 新しいファイルが`.vcxproj`に追加されている
- [ ] 新しいファイルが`.vcxproj.filters`の適切なフィルターに追加されている
- [ ] シェーダーファイルなどのアセットが適切に設定されている

### Git
- [ ] `.gitignore`で不要なファイルが除外されている
- [ ] ビルド成果物（`.obj`, `.exe`など）がコミットされていない

## 5. Performance (Optional)

パフォーマンスが重要な場合は以下も確認：

- [ ] 不要なコピーや動的メモリ確保がない
- [ ] 適切にconst参照を使用している
- [ ] Hot pathで不要な処理が行われていない

## 6. User Communication

### Task Completion Report
ユーザーに報告する際は以下を含める：

- [ ] 実装した機能の概要
- [ ] 変更/追加されたファイルのリスト
- [ ] 使い方や統合方法の簡単な説明
- [ ] 既知の制限事項や今後の改善点（あれば）

## 7. Future Considerations

今後のタスクのために：

- [ ] TODO コメントで将来の改善点を記録
- [ ] 拡張性を考慮した設計になっている
- [ ] テストケースの追加が必要かどうかメモ

---

**Note**: すべてのチェック項目が必須ではありません。タスクの性質に応じて適切な項目を選択してください。
